# Pedagogical Chain-of-Thought Prompting for AI-Generated Algebra Feedback

This repository contains the experimental code, prompt templates, case data, generated outputs, and LLM-as-judge evaluation files for a study on AI-generated feedback for algebra errors.

The project evaluates how different prompting strategies affect the quality of feedback generated by large language models for student algebra mistakes, with a particular focus on Pedagogical Chain-of-Thought prompting and Taglish localization.

## Study Overview

The study investigates whether prompt engineering can improve the quality of AI-generated instructional feedback for algebra error cases. It compares multiple prompting strategies across different language models and evaluates the generated feedback using both model-based and human-oriented evaluation dimensions.

The work was conducted as part of an undergraduate Special Problem in Computer Science at the University of the Philippines Cebu.

## Research Focus

The study examines how different prompting strategies influence feedback quality across dimensions such as:

* Diagnostic accuracy
* Pedagogical scaffolding
* Linguistic naturalness
* Actionability
* Teacher authenticity

The experiment compares feedback generated by different models and prompt designs for a fixed set of algebra error cases.

## Prompting Strategies

The study includes the following prompting strategies:

* Zero-Shot prompting
* Standard Chain-of-Thought prompting
* Pedagogical Chain-of-Thought prompting
* Taglish Pedagogical Chain-of-Thought prompting

## Models Used

The experiment used generated outputs from:

* Gemini
* Llama

The repository also includes scripts for generating, resuming, and evaluating model outputs.

## Experimental Workflow

The general workflow of the experiment was:

1. Prepare algebra error cases.
2. Define prompt templates for each prompting strategy.
3. Generate feedback outputs using Gemini and Llama.
4. Save generated outputs in CSV and JSON formats.
5. Evaluate outputs using the LLM-as-judge script.
6. Compare output quality across prompting strategies and models.

## How to Use

Clone the repository:

```bash
git clone https://github.com/cess-m/pcot-algebra-feedback-study.git
cd pcot-algebra-feedback-study
```

Create and activate a Python environment:

```bash
python -m venv .venv
.venv\Scripts\activate
```

Install required dependencies based on the scripts you plan to run. If using model APIs, configure the required API credentials locally.

Run a generation script:

```bash
python run_gemini_v3.py
```

or:

```bash
python run_llama_v3.py
```

Run LLM-as-judge evaluation:

```bash
python llm_judge.py
```
